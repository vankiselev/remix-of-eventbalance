
Цель: исправить синхронизацию авансов между админом и сотрудником и обновить UX карточки авансов (всегда показывать блок у сотрудника, слева от налички, с именем того, кто выдал аванс).

1) Диагностика проблемы (что уже видно по коду и сети)
- Выдача аванса сейчас реально сохраняется (PATCH в `profiles` успешный).
- Данные “мой аванс” читаются только по текущему `auth`-пользователю (`useMyAdvance`), без режима просмотра выбранного сотрудника.
- В `RealtimeSync` на изменения `profiles` не инвалидируются ключи `['all-advances']` и `['my-advance']`.
- Карточка сотрудника скрывается полностью, если сумма 0 (`return null`), из-за чего пропадает “левый блок” и визуально кажется, что раздела нет.
- Текст “Выдано администратором” захардкожен, имени выдавшего нет в данных.

2) Изменения в данных (миграция БД)
- Добавить в `profiles` поля:
  - `advance_issued_by uuid null` (FK на `public.profiles(id)`, `ON DELETE SET NULL`)
  - `advance_issued_at timestamptz null`
- (Для живой синхронизации) добавить `public.profiles` в realtime publication (`supabase_realtime`) через идемпотентный SQL-блок.
- Никаких изменений в auth-таблицах/резервных схемах.

3) Обновить логику записи аванса
- `AdvanceEditDialog.tsx`:
  - при сохранении суммы > 0 писать:
    - `advance_balance`
    - `advance_issued_by = текущий пользователь`
    - `advance_issued_at = now()`
  - при сумме 0 очищать:
    - `advance_balance = 0`
    - `advance_issued_by = null`
    - `advance_issued_at = null`
- `AdvancesSummaryCard.tsx` (кнопка удаления):
  - тоже обнулять и очищать metadata тем же способом.
- После мутаций оставляем инвалидацию `['all-advances']`, `['my-advance']`, `['profiles']`.

4) Обновить hooks авансов
- `useMyAdvance` расширить: принимать опциональный `targetUserId` (для просмотра сотрудника админом) и возвращать объект:
  - `amount`
  - `issuedById`
  - `issuedByName`
  - `issuedAt`
- Если `issuedById` есть — подтягивать имя из `profiles` (или через relation после FK).
- Добавить более агрессивное обновление для этого кейса:
  - `refetchOnMount: "always"` (или `staleTime: 0`) для исключения “залипания” кэша.
- `useAllAdvances` оставить для админ-сводки (по необходимости можно добавить issuer-поля позже).

5) Обновить UI карточки авансов
- `AdvancesSummaryCard`:
  - добавить режим “employee view” даже для админа при просмотре конкретного сотрудника (`employeeId` проп).
  - у сотрудника карточку показывать всегда (не скрывать при 0), чтобы блок “Авансы” всегда был слева.
  - если сумма > 0: показывать сумму + строку вида `Выдал(а): Киселёв Иван`.
  - если сумма = 0: показывать нейтральный статус “Аванс пока не выдан”.
  - заменить формулировку “Выдано администратором” на имя выдавшего (с fallback, если данных нет).
- `Finances.tsx`:
  - в режиме выбранного сотрудника передавать `employeeId={selectedEmployee.id}` в `AdvancesSummaryCard`, чтобы показывался именно аванс этого сотрудника.
  - сохранить порядок блоков: авансы слева, наличка справа.

6) Глобальная синхронизация между сессиями
- `App.tsx` (`RealtimeSync`):
  - в обработчике изменений `profiles` добавить:
    - `invalidateQueries(['all-advances'])`
    - `invalidateQueries(['my-advance'])`
- Это даст автообновление у сотрудника, даже если админ выдал аванс из другой сессии.

7) Проверка после внедрения
- Сценарий 1: админ выдает 10 000 сотруднику → у сотрудника без перезагрузки появляется сумма и имя выдавшего.
- Сценарий 2: админ удаляет аванс → у сотрудника карточка остается, но показывает “Аванс пока не выдан”.
- Сценарий 3: админ открывает профиль конкретного сотрудника в финансах → в левом блоке данные именно этого сотрудника.
- Сценарий 4: порядок блоков в верхней зоне сохранен (авансы слева, наличка справа).
